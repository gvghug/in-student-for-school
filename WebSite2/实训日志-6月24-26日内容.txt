【实训总结】（填入模板"实训总结"栏，包含问题+解决方案+明日计划）

遇到问题：
（1）在账簿管理模块中，当前余额字段（currentBalance）需要根据后续记账流水动态计算，直接通过 GridView 绑定静态数据库字段无法反映实时余额，导致列表页显示的余额与实际不符。
（2）收支项目管理中，RadioButtonList 控件的 SelectedValue 获取的值是字符串类型，而数据库 itemType 字段设计为 INT 类型，直接赋值导致数据类型不匹配的异常。
（3）多个 .aspx 页面中 SqlDataSource 的 ConnectionString 属性重复配置，修改连接字符串时需要逐个页面更新，维护不便。

解决方案：
（1）在 GridView 的 RowDataBound 事件中通过代码动态计算当前余额列的值，或者先展示初始余额，待记账模块完成后通过 SQL 聚合查询实时计算。
（2）在 btn_save_Click 中使用 int.Parse(itemTypeRadio.SelectedValue) 将字符串显式转换为整型后再赋值给 SqlCommand 参数。
（3）将数据库连接字符串统一配置在 Web.config 的 connectionStrings 节点中，所有 SqlDataSource 使用 <%$ ConnectionStrings:FMSDB %> 表达式引用，实现一处修改全局生效。

遇到问题：
（1）ASP.NET Chart 控件在运行时提示"ChartImg.axd"找不到或 HTTP 404 错误，饼图和柱状图无法正常渲染显示。
（2）报表查询时，当用户未选择任何筛选条件直接点击查询，动态拼接的 SQL 出现"WHERE AND"语法错误（WHERE 关键字后直接跟 AND 导致 SQL 非法）。
（3）删除操作没有任何提示，用户误点删除按钮后数据直接被删除无法恢复，存在操作风险。

解决方案：
（1）在 Web.config 的 system.web/httpHandlers 节点中添加 ChartImg.axd 的 HTTP 处理程序映射配置：<add path="ChartImg.axd" verb="GET,HEAD,POST" type="System.Web.UI.DataVisualization.Charting.ChartHttpHandler, System.Web.DataVisualization"/>，并在 system.webServer/handlers 中同步添加。
（2）使用 StringBuilder 拼接 SQL 时引入布尔标志位 hasCondition，首次添加条件时使用"WHERE"，后续条件使用"AND"，或者使用 WHERE 1=1 作为基准条件再追加 AND 条件，确保 SQL 语法始终正确。
（3）为所有 ImageButton 删除按钮添加 OnClientClick="return confirm('确定要删除该记录吗？删除后不可恢复！');" 属性，利用 JavaScript 确认框拦截误操作。

【实训总结】（填入模板"实训总结"栏，包含问题+解决方案+明日计划）

遇到问题：
（1）在实现退出登录时发现，仅跳转到 login.aspx 页面并不安全——用户点击浏览器后退按钮仍可回到主页面继续操作，因为 Session 未被清除，服务器仍认为用户处于登录状态。
（2）IIS 部署测试时出现"无法打开登录所请求的数据库"错误，提示 Login failed for user 'IIS APPPOOL\DefaultAppPool'，应用程序池的身份无法访问 SQL Server 数据库。
（3）项目报告的字数要求较高，需要详细描述技术细节和心得，但时间较紧需要高效完成。

解决方案：
（1）创建专门的 logout.aspx 中间页面，在 Page_Load 中调用 Session.Clear() 清空所有 Session 键值对，再调用 Session.Abandon() 终止当前会话并触发 Session_End 事件，最后跳转到 login.aspx。这样即使点击后退按钮，Session 已失效，再次访问受保护页面时会被重定向到登录页。
（2）在 SQL Server Management Studio 中为 NT AUTHORITY\NETWORK SERVICE 或 IIS APPPOOL\DefaultAppPool 创建登录账户并授予 FamilyFinanceSystem 数据库的 db_datareader 和 db_datawriter 角色权限；或者将 Web.config 连接字符串改为 SQL Server 混合认证模式，使用 sa 或专用账户登录。
（3）合理规划报告结构，先列出大纲再逐段填充，将之前五天的实训日志内容整合提炼到报告中，代码部分直接截取项目中的关键代码段并添加注释说明。
